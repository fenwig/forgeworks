The Dashboard is the main navigation hub for TI Forgeworks. All other modules load inside it as embedded pages — your sidebar and header stay persistent across the entire app.

Navigation

Click any entry in the left sidebar to load that module. The active module is highlighted in gold. You can also click the module cards on the home screen to launch any section directly.

The Sync Data button in the header bar is a shortcut that takes you directly to Data Sync, regardless of which module is currently open.

Import / Export

The Import / Export button in the header opens a backup panel. Use this to save or restore your entire dataset — material lots, blueprints, tracking flags, orders, and crafting history.

  • Export — downloads a single JSON file containing all app data. Store it somewhere safe. The export date is shown next to the button.
  • Import — paste or load a previously exported JSON file to restore your data. All current data is replaced.

Tip: Export your data before running a Data Sync if you want a fallback, or any time you make significant changes to your inventory or orders.

Status Indicator

The SYSTEMS ONLINE pill in the header center confirms that the app has loaded correctly. It has no interactive function.

The Material Database is your central inventory ledger. Every ore and gem lot you own is recorded here with its quality, quantity, and storage location. All other modules read from this data to check crafting readiness.

Adding a Lot

Click + Add Lot to open the entry form. Select the ore or gem, enter a quality value (1–1000) and a quantity, then choose a storage location.

  • Ore quantities are displayed in SCU throughout the app, but entered in cSCU (centri-SCU). One SCU = 100 cSCU. The form label shows "Quantity (cSCU)" as a reminder.
  • Hand-mined gems (Hadanite, Janalite, Dolivine, etc.) are entered as individual gem counts, not cSCU. The form label updates automatically when a gem is selected.
  • The location defaults to Levski. Type a custom location or select one you have saved previously.

Shortcut: Use OCR Input to import refinery yields directly from a screenshot, or Add Bulk Material to enter many lots of the same ore in rapid sequence.

Filters and Sorting

The filter bar lets you narrow the list by Type (All / Ore / Gems) and Quality band (All / 700s / 800s / 900s). Use the search box to filter by ore name or location. Click any column header to sort.

Editing and Deleting
  • QTY — click to edit the quantity of a lot inline. Press OK to confirm or ✕ to cancel.
  • DEL — immediately removes the lot. There is no undo.
Bulk Operations

Check the box on one or more rows to select them. Three operations become available:

  • Combine — merges two or more lots of the same ore at the same location into a single lot. The resulting quality is a weighted average by quantity. Requires all selected lots to share the same ore and location.
  • Split — divides one selected lot into two. Enter how many cSCU to split off; the remainder stays as a second lot. Both halves keep the original quality.
  • Move — reassigns all selected lots (at the same current location) to a new location.
Managing Locations

The location dropdown in both the Add Lot form and the Move dialog includes a Custom... option and an — Edit List — option. Custom locations you add are saved and shared across all modules. A location that still has lots assigned to it cannot be removed.

Open Edit List to set your default storage location. The current default is marked with a gold DEFAULT badge. Click Set Default on any location — including Levski — to make it the new default. The change applies immediately across all modules. If the default location is removed, it resets to Levski automatically.

Statistics Footer

The header shows total lot count, total cSCU across all lots, and gem vs ore subtotals. The footer shows how many lots are currently visible vs total, the cSCU sum of visible lots, and the quantity of any selected rows.

Resource Acquisition is designed for rapid data entry when you have a batch of lots from a single mining or refinery session. You pick one ore, enter multiple quality/quantity pairs in quick succession, then commit them all at once.

Workflow
  1. Select a material — use the All / Gems / Ore filter buttons to narrow the list, then pick from the dropdown. The selected ore locks in and is shown in the panel.
  2. Set a location — choose from the dropdown or enter a custom location. All lots in this session will use this location.
  3. Enter each lot — type quality / quantity in the entry field and press Enter (or click + ADD). Repeat for each lot.
  4. Review the queue — each entry appears in the Session Queue below. Click ✕ on any row to remove it before committing.
  5. Commit to Inventory — click the gold button to write all queued lots to the Material Database. The queue clears and a confirmation toast appears.

Format: Quality / Quantity cSCU or Gems  ·  Example: 875 / 45 = quality 875, 45 cSCU. For gems, quantity is a gem count, not cSCU. The input field placeholder reminds you of the format.

Notes
  • You can switch between different ores mid-session by changing the material dropdown — each lot in the queue retains the ore it was entered with.
  • Clear Queue discards all staged entries without saving. Committed lots cannot be un-committed from this screen — use Material Database to delete them if needed.
  • For refinery box yields specifically, use OCR Import instead — it auto-splits yields into individual box lots.

The Blueprint Database is your full reference for every craftable item in the game. It has three tabs: a browser for exploring and tagging blueprints, a tracker for monitoring crafting readiness, and a material needs calculator.

Blueprint Browser

Displays all blueprints loaded from Data Sync. Use the filter row to narrow by type, ownership, and search by name. Clicking a blueprint row expands it to show its ingredient list.

ControlWhat it does
Type filterShow All, Armor, Weapon, or Ammo. Ammo is hidden from the All view — it only appears when you explicitly select the Ammo filter.
All Blueprints / Owned OnlyToggle to show only blueprints you have marked as owned.
Mission OnlyFilters to blueprints that require a mission unlock.
Armor filtersVisible when Armor type is active. Filter by Category, Slot (Helmet / Core / Legs / Arms / Backpack), and Weight (Light / Medium / Heavy).
Weapon filtersVisible when Weapon type is active. Filter by weapon type (Rifle, SMG, Pistol, etc.).

Note: Armor Category (Combat, Cosmonaut, Engineer, etc.) is not available from the game API. Set it manually by opening a blueprint and editing its category field after syncing.

Crafting Tracker

Shows all blueprints you have personally tracked, plus any that have an active order attached. Each row displays your selected quality tier badges and whether your current inventory can cover them.

  • Tier badges (BASE / 700+ / 800+ / 900+) — click to toggle which tiers you are planning to craft. A ✓ appears when your inventory has sufficient materials at that quality band. BASE tracks quality 500–699 materials (for lower-tier crafts). If you specify no tiers at all, the blueprint accepts any quality material — material readiness checks are disabled.
  • Order pills — shown automatically for any blueprint with an active order. Turn green when materials are sufficient for the order's quality tier.
  • FORGE → — appears when at least one tier is ready. Clicking it opens The Forge with that blueprint pre-selected on the workbench.
  • Expand row — shows a per-ingredient breakdown: how much you need vs. how much you have in each quality band.
  • ✕ button — removes the blueprint from personal tracking. Order-linked blueprints remain until the order is delivered or deleted.

Aslarite rule: Aslarite is a universal ingredient. Any aslarite at quality 500 or above counts toward requirements — it is not band-locked like other materials.

Material Needs

Aggregates ingredient requirements across all tracked blueprints and active orders. Shows what you need per quality band vs. what you currently have in inventory.

  • NEED column — the quantity required per ingredient per quality tier, based on blueprints you are tracking at that tier.
  • HAVE column — your current inventory, color-coded: green = sufficient materials to fulfill the need, red = shortage.
  • Personal tracking contributes to whichever tiers you have selected on that blueprint in the Tracker tab. Orders contribute only to their specific quality band.
  • You can toggle quality tiers in the Tracker tab to focus the Material Needs display on specific bands (e.g., tracking only 800+ and 900+ will show needs only for those tiers).
Ship Components (4.8+)

Components work identically to armor and weapons in the tracker and material needs view. To focus on components:

  • Use the Type filter in the Blueprint Browser to select Components and show only component blueprints.
  • Track components the same way as armor/weapons — click TRACK and toggle your desired quality tiers (BASE / 700+ / 800+ / 900+).
  • Component tracking contributes to the Material Needs aggregation just like any other blueprint.
  • All material needs calculations include component ingredients alongside armor and weapon materials.

The Forge is where you execute crafting jobs. It walks you through selecting material lots for each ingredient slot, projects the resulting quality tier, deducts consumed materials from your inventory, and logs the completed craft.

Blueprint Browser

The first tab shows all blueprints marked as Owned in Blueprint Database. Use the type filter or the Tracked / Orders quick filters to narrow the list. Click Craft This on any blueprint to load it onto the workbench.

You can also arrive here directly from Blueprint Database — the FORGE → button on a tracked blueprint pre-selects it on the workbench automatically.

Craft Workbench

The workbench shows one slot per ingredient in the blueprint. For each slot:

  • Click Add Lot to open the lot picker for that ingredient. Available lots are shown with quality, quantity, and location.
  • Click High → Low / Low → High in the picker to sort by quality.
  • Click Select on a lot to assign it. The module takes only what is needed — if the lot has more than required, only the required amount is reserved.
  • Multiple lots can be used to fill one slot if a single lot is not large enough.
  • Use Remove Last to undo the most recent lot selection for a slot.

A progress bar fills as each slot is satisfied. When all slots are filled, the Projected Quality Tier appears at the bottom.

Quality Tier Calculation

The projected tier is based on the weighted average quality of all non-aslarite ingredients. Aslarite does not affect quality. The result maps to a tier: 700+ (700–799), 800+ (800–899), or 900+ (900+).

If an order is attached and the projected tier is below the order's target quality, the Make Item button is blocked with a warning. You cannot accidentally under-craft for an attached order.

Attaching an Order

Click Attach Order to link an active order to this craft. Only orders for the current blueprint are shown. When an order is attached, the target quality tier is displayed as a reminder. After crafting, the linked order is automatically advanced to Ready status.

Making the Item

Click Make Item when all slots are filled. The module:

  • Deducts used quantities from Material Database lots (lots emptied to zero are removed).
  • Records a crafting job in the Crafting Log with all ingredient details and the resulting quality tier.
  • Advances any attached order to Ready.
  • Shows a result summary on the workbench with material quality and crafting property effects.
Crafting Log

The third tab shows all past crafting jobs, newest first. Filter by All / Personal / Orders. Expand any entry to see per-material quality averages and crafting property effects. You can also attach or detach orders from log entries retroactively.

Crafting properties show the stat effect each material contributed based on its quality. Energy weapon properties are currently pending — they display as "pending game data" until extracted.

The Order Tracker manages customer crafting requests from intake to delivery. Orders are tracked through a three-stage pipeline and automatically link to Blueprint Database and The Forge so material readiness is always visible.

Creating an Order

Go to the New Order tab and fill in the form:

  • Customer — the player handle or name the order is for.
  • Blueprint — select from all blueprints loaded in Data Sync.
  • Quality Tier — choose 700, 800, or 900. This determines which quality band of materials is required.
  • Deadline — optional. Deadlines within 3 days turn amber; overdue deadlines turn red.
  • Trade Agreed — free text for payment terms (aUEC amount, items, or other arrangements).
  • Notes — any special instructions or delivery details.

Clicking Create Order saves the order and automatically adds the blueprint to Crafting Tracker so its material readiness is monitored.

Expand any order row to see its full details and action buttons. Use Mark Ready / Mark Delivered to advance the status, or the revert button to step back. A MAT ✓ badge appears on orders where current inventory is sufficient.

Active Orders Filter

The filter bar on the Active Orders tab lets you view All Active, In Progress only, or Ready only. Delivered orders are always in the Past Orders tab.

Deleting an Order

Expand an order row and click Delete Order. You must confirm. Deleting an order removes it permanently and clears it from Crafting Tracker (unless the blueprint is still personally tracked).

Note: Past Orders are not saved across page reloads. Delivered orders are visible in the current session only. Use Reports → Orders Filled for a persistent record.

Reports generates shareable, branded summaries of your inventory, blueprint availability, and crafting history. Each report is rendered on a light background formatted for readability and screenshotting. Ammunition blueprints are excluded from all reports.

Report 1 — Materials

A snapshot of your current Material Database inventory. Shows all ore and gem lots grouped by quality band, with quantities in cSCU. Use the date range filter to limit which lots are shown by when they were added. The Personal / Order checkboxes let you scope the report to specific tracking types.

Report 2 — Crafting Availability

Shows which blueprints can currently be crafted and at which quality tiers, based on your live inventory. Armor is grouped by set with slot pills; weapons are listed individually. A blueprint appears in the 700+, 800+, or 900+ column only when you have enough material at that quality band.

  • Use the Armor / Weapons toggle to switch between item types.
  • Only blueprints loaded from Data Sync are included. Run a sync first if this report is empty.
  • Quality determination uses the Core piece for armor sets.
Report 3 — Owned Blueprints

Lists all blueprints you have marked as Owned in Blueprint Database, sorted alphabetically. Useful for sharing your schematic inventory with org members or customers.

Report 4 — Orders Filled

A record of delivered orders. Shows customer, blueprint, quality tier, trade agreed, and delivery date. Filter by date range to scope to a specific period.

Report 5 — Crafted Items

A full log of all crafting jobs from The Forge. Shows item name, quality tier, and the materials used with their average quality. Useful for reviewing craft history and verifying what was made for which order.

OCR Import reads your Star Citizen refinery yield screen directly and extracts lot data automatically, eliminating manual entry for large batches of refined ore.

Setup — First Use

Configure the import settings in the setup bar at the top before capturing:

  • Storage Location — where the imported lots will be stored in your inventory.
  • Date — defaults to today. Change if you are entering historical data.
Capturing and Cropping

Click Start OCR to open the capture picker. In the picker, click the Window tab, then select Star Citizen from the list. Once selected, a snapshot appears on the canvas.

Click and drag on the canvas to draw a yellow overlay region around the yield table — specifically the rows showing ore names, box sizes, and quantities.

  • OCR runs automatically as soon as you finish drawing the region. There is no separate button to click.
  • Your crop region is saved and reused on your next visit, so you only need to draw it once per screen layout.
Result Indicators

Each ore name in the results table is matched against a known list. Three outcomes are possible:

OutcomeIndicator
Exact matchNothing shown — ore name accepted silently.
Fuzzy match
(close but not exact)		Gold ~ badge to the right of the ore name. Hover to see what OCR originally read.
No matchAmber ? badge. Hover shows "No match found — please verify." Edit the field manually.

All fields remain editable regardless of match status. Fuzzy matching catches common OCR errors such as transposed or missing letters.

Reviewing Results

Check each row before committing. You can edit any field directly in the table or remove rows that should not be imported. The module automatically splits refinery yields into standard box sizes (3200 / 1600 / 800 / 400 / 200 / 100 cSCU), matching how the game fills boxes largest-first.

Committing

Click Commit to Inventory to write the reviewed lots to Material Database. A success message confirms the count.

Best practices: The capture can only read what is currently visible on screen. Refine no more than 10 items at a time for reliable results. Any ore that falls below the scroll line will not be captured — add those lots manually via Material Database or Resource Acquisition. Do this before collecting the order.

Data Sync pulls the latest blueprint data from the Star Citizen Wiki community API and stores it locally. This is how the app knows what can be crafted, what ingredients are required, and which blueprints need a mission unlock.

When to Sync
  • Run a sync once after installing the app to populate the Blueprint Database.
  • Run again after each game patch that adds or changes craftable items.
  • The header shows the date and game version of your last sync, and how many blueprints are currently loaded.
What Gets Imported
  • Armor, weapon, ammunition, and ship component blueprints — approximately 1,548 items after filtering and deduplication.
  • Ingredient names and quantities (converted to cSCU).
  • Mission unlock flags.
  • Weapon type, armor slot information, and component metadata (type, class, grade, size).

Excluded types: weapon attachments, liveries, and other non-craftable entries. These are filtered automatically and counted in the sync log as skipped.

Duplicate Prevention

The Star Citizen Wiki API contains 11 known duplicate blueprint entries. TI Forgeworks automatically blocks these duplicates on every sync to prevent them from reappearing after game patches or re-syncs. Your owned and tracked flags are preserved even when duplicates are filtered out, so you never lose your crafting data.

What Is NOT Imported
  • Armor category (Combat, Cosmonaut, Engineer, etc.) — not available from the API. Set this manually in Blueprint Database after syncing.
  • Crafting properties — stat effects per material are stored in a local file and calculated independently of the API.
Ownership and Tracking Flags

Your Owned and Tracked flags are preserved across syncs. Each blueprint is matched by its stable Wiki UUID, so a patch-day re-sync will not reset your ownership data. Only blueprints that are removed entirely from the game will lose their flags.

Running a Sync

Click Sync Blueprints. The module makes approximately 11 API requests in sequence, showing progress for each page. When complete, a results summary shows how many blueprints were imported, updated, and skipped. The sync history panel below records the last 20 syncs.

After syncing: Open Blueprint Database and mark your owned blueprints. If you craft for customers, also check that any mission-locked blueprints you own are flagged correctly.

API Status

The status card shows the Wiki API connectivity check on load. If it shows an error, your internet connection may be unavailable, or the Wiki API may be temporarily down. Syncing is blocked when the API is unreachable.

Starting in the 4.8 patch, TI Forgeworks will support crafting of ship components alongside armor and weapons. Components are categorized by type, class, grade, and size, with their own crafting recipes and quality tier requirements.

Component Types

Each component belongs to one of ten functional types:

TypeDescription
CoolerThermal management systems. Higher quality reduces ship heat signature.
Mining LaserMining extraction tools. Quality affects extraction efficiency and yield.
Power PlantCore energy generation. Quality improves power output and efficiency.
Quantum DriveJump drive systems. Quality affects jump range and charge speed.
RadarSensor and detection systems. Quality improves detection range and accuracy.
Refuelling ProbeIn-flight refueling equipment. Quality affects transfer speed and range.
Salvage EquipmentShip salvage extraction gear. Quality affects processing speed and yields.
Shield GeneratorProtective energy barriers. Quality improves shield strength and recharge.
Ship WeaponMounted offensive systems. Quality affects damage output and fire rate.
Tractor BeamObject manipulation systems. Quality affects lift capacity and range.
Component Classes

Each component is manufactured in one of five classes, reflecting power level and intended use:

ClassPurpose
CivilianEntry-level, everyday use. Lowest material costs.
MilitaryCombat and high-stress applications. Moderate material costs.
IndustrialProfessional extraction and processing. Higher material costs.
StealthSignature reduction and specialized scanning. Premium material costs.
CompetitionRacing and extreme performance. Highest material costs, rare ingredients.
Component Grades

Within each class, components are graded A through D (best to worst). Grades reflect material quality and manufacturing precision:

  • Grade A — Premium components. Requires highest-quality materials.
  • Grade B — High-quality. Requires quality-tier materials.
  • Grade C — Standard. Requires mid-tier materials.
  • Grade D — Economic. Uses lower-quality materials.
Component Sizes

Ship components scale in size to fit different vessel classes. Size 0 is smallest; Size 4 is largest. A component size must match the ship's hardpoint size, or it will not fit.

  • Size 0 — Aurora, Mustang, 100 series.
  • Size 1 — Drake Cutlass, Avenger, Reliant.
  • Size 2 — Constellation, Prospector, Sabre, Vanguard.
  • Size 3 — Carrack, Hammerhead, Reclaimer.
  • Size 4 — Bengal, Idris, capital-class ships.
Quality Tiers and Crafting

Component crafting follows the same quality tier system as armor and weapons:

  • 500+ tier — Basic crafted components (new in 4.8).
  • 700+ tier — Refined components.
  • 800+ tier — High-quality components.
  • 900+ tier — Premium components.

Higher tiers require more materials and rarer ingredients, but deliver superior performance stats in game.

Tracking and Orders

Ship components integrate into all existing Forgeworks features. You can:

  • Own and track component blueprints in Blueprint Database.
  • View component crafting availability in Reports (Crafting Availability).
  • Create orders for components in Orders module.
  • Log component crafting in Crafting Log.
  • Monitor component material requirements in Material Needs.

Tip: Components often use the same core materials as armor and weapons, so building material reserves helps across all three crafting categories.

Sync with 4.8 Patch

When the 4.8 patch releases, run a Data Sync to import all available component blueprints. The sync will populate component entries automatically — no manual configuration needed. You can then mark components as owned and start tracking them immediately.
